## GT Animation: Frame Interpolation

GT uses **morph target animation** with identity-matrix weight encoding:
- Stores vertex position deltas for frames 0, 4, 8, … (every 4th frame)
- GLTF animation sampler uses LINEAR interpolation
- Three.js automatically blends between adjacent stored frames at runtime — visually close to full-frame playback

Bones uses **skeletal TRS animation**; Three.js handles SLERP/LERP automatically.

## Floor Tracking: Keeping Feet on the Pedestal

### Why `Box3.setFromObject` Fails for Morph Targets

`Box3.setFromObject` unions the bounding boxes of **all morph states** regardless of current weights — it always returns the worst-case expanded box. Useless for per-frame floor tracking.

### Solution: Per-Frame CPU Vertex Iteration

Each frame, compute the actual minimum Y using the current morph weights:

```js
// Formula: y_v = base_Y[v] + Σ(weight[m] × delta_Y[m][v])
let frameLocalMinY = Infinity
for (let v = 0; v < _predPos.count; v++) {
  let y = _predPos.getY(v)
  for (let m = 0; m < _predWeights.length; m++) {
    if (_predWeights[m] > 1e-6) y += _predWeights[m] * _predMorphPos[m].getY(v)
  }
  if (y < frameLocalMinY) frameLocalMinY = y
}
predModel.position.y = initialOffsetY - (frameLocalMinY - baseLocalMinY) * floorScaleY
```

### Key Parameters (computed by `computePedestalTransform` in `loader.js`)

```
initialOffsetY = 1.6 - sMinY × scale          // base-pose floor in parent-local space = 1.6
baseLocalMinY  = min Y of base pose in mesh local space
floorScaleY    = scale × parentSizeY / localSizeY
```

Note: bounding box is computed in **parent-local space** (not world space) and includes the display rotation (`MODEL_ROTATION_Y = -15°`). See `docs/PROGRESS.md` for details on coordinate space fixes.

Mathematically guaranteed: `world_floor_Y = 1.6` for every frame, regardless of morph state.

### Bones Floor Tracking

Bones uses TRS animation (no morph targets), so `Box3.setFromObject` correctly reflects the current pose:

```js
function updateBonesFloor() {
  bonesModel.position.y = 0           // zero out last frame's offset before computing box
  _bonesBox.setFromObject(bonesModel)
  bonesModel.position.y = 1.6 - _bonesBox.min.y
}
```

## GT ↔ Bones Frame Sync

When toggling views, use **normalized time mapping** so GT frame N maps to Bones frame N:

```js
// Switching to Bones
const norm = predAction.time / predAction.getClip().duration
bonesAction.time = norm * bonesAction.getClip().duration

// Switching back to Mesh
const norm = bonesAction.time / bonesAction.getClip().duration
predAction.time = norm * predAction.getClip().duration
```

Use `action.time` (wraps within clip duration under `LoopRepeat`), not `mixer.time` (monotonically increasing — overflows after first loop).
